Make the tasks.dante.id Kanban dashboard fully responsive for mobile and tablet.

## Tech Stack
- React 19, Vite, Tailwind CSS (imported in index.css)
- All components use inline styles (no CSS modules)
- Material Design 3 color scheme via CSS variables in index.css
- No component library — all custom components

## Files to Modify
1. client/src/index.css — Add media queries, responsive utilities
2. client/src/App.jsx — Responsive layout: tab-based column switching on mobile, grid on tablet
3. client/src/components/Column.jsx — Full-width on mobile

5. client/src/components/TaskDetailModal.jsx — Full-screen sheet on mobile (not centered modal)

7. client/src/components/StatsBar.jsx — Wrap/scroll on small screens
8. client/src/components/ChatPanel.jsx — Full-width on mobile

## Step 1: Create useBreakpoint hook
Create client/src/hooks/useBreakpoint.js:
```js
import { useState, useEffect } from 'react';
export default function useBreakpoint() {
  const [width, setWidth] = useState(window.innerWidth);
  useEffect(() => {
    const handler = () => setWidth(window.innerWidth);
    window.addEventListener('resize', handler);
    return () => window.removeEventListener('resize', handler);
  }, []);
  return { isMobile: width < 640, isTablet: width >= 640 && width < 1024, isDesktop: width >= 1024 };
}
```

## Mobile Requirements (< 640px)
1. Show one column at a time. Add a sticky bottom nav bar (Material 3 style: white bg, border-top, 56px height) with tabs: Todo | Active | QA | Done | Failed. Tapping a tab switches which column is visible. No horizontal scroll.
2. FAB button (bottom-right, 16px above bottom nav): 56px, border-radius 16px, --md-primary bg (#6750A4), white "+" icon, box-shadow. Opens DispatchModal. Hide the desktop "+ New Task" header button.
3. TaskDetailModal: full-screen sheet. position:fixed, inset:0, white bg, rounded top corners 24px, slide-up feel. Close via X button at top-right. No centered modal overlay.
4. DispatchModal: also full-screen on mobile, same treatment.
5. Header: simplified — just logo/title on left. Hide StatsBar on mobile.
6. Filter bar: flex-wrap so chips wrap to multiple lines. Project selector full-width.
7. TaskCard buttons: min-height 44px for touch targets.
8. ChatPanel: full-screen overlay (position:fixed, inset:0, z-index:1000) on mobile instead of fixed 400x500.

## Tablet Requirements (640px - 1024px)
1. Show 2-3 columns visible with horizontal scroll for rest (use flex with overflow-x: auto, min-width ~280px per column)
2. StatsBar: compact, wraps if needed (flex-wrap)
3. Modals: centered, width 90%, max-width 600px

## Desktop (> 1024px)
- NO changes to current layout

## Implementation Notes
- Import useBreakpoint in App.jsx. Conditionally render mobile layout (single column + bottom nav + FAB) vs desktop layout (current).
- In mobile layout: use useState for activeColumn (default 'todo'). Map column statuses to tab labels.
- Pass isMobile prop to modal components so they can switch between full-screen and centered.
- Use CSS media queries in index.css for things that can be done purely in CSS (font sizes, padding).
- For layout switching (showing/hiding columns, bottom nav), use the hook since styles are inline.
